# Executive Dashboard

The **Executive Dashboard** provides a high-level strategic overview designed for CXOs, Directors, and Secretary-level officials. It aggregates data across all projects and corridors to present a single view of the entire infrastructure programme.

![Executive Dashboard](/img/screenshots/11-dashboard-executive.png)

## Key Sections

### Corridor Overview

A summary of all infrastructure corridors with key metrics:

| Metric | Description |
|---|---|
| **Total Projects** | Number of projects assigned to each corridor |
| **Total Budget** | Aggregate sanctioned budget across all projects |
| **Overall Progress** | Weighted average physical progress percentage |
| **Status Distribution** | Breakdown of projects by status (Planned / Active / Completed) |

### Key Performance Indicators (KPIs)

High-level KPIs aggregated across the portfolio:

| KPI | What It Measures |
|---|---|
| **Portfolio CPI** | Cost Performance Index across all active projects — values below 1.0 indicate cost overrun |
| **Portfolio SPI** | Schedule Performance Index — values below 1.0 indicate schedule slippage |
| **Budget Utilization** | Percentage of total sanctioned budget that has been spent |
| **On-Time Delivery** | Percentage of milestones delivered on or before their planned date |

### Top Risks Summary

A table of the highest-severity risks across the entire portfolio, showing:

- **Risk description** and category
- **Probability** and **Impact** ratings
- **Risk owner** and **mitigation status**
- **Affected project(s)**

### Project Status Overview

A visual summary (chart or table) showing how many projects fall into each status category and their trend over time.

---

## When to Use This Dashboard

- **Monthly steering committee meetings** — present programme-level health at a glance
- **Risk escalation reviews** — identify the top risks demanding executive attention
- **Budget reviews** — see how portfolio spending compares to sanctioned amounts
- **Strategic decisions** — determine which corridors or programmes need intervention
