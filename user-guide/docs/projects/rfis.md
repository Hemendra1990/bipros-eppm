# RFIs (Requests for Information)

A **Request for Information (RFI)** is a formal question raised during a project when clarity is needed on drawings, specifications, scope, or contractual matters. The RFI module tracks these requests from submission through response and closure.

## Accessing RFIs

Navigate to a project, then select **RFIs** from the **More** dropdown menu.

![RFIs](/img/screenshots/27-project-rfis.png)

## RFI Table

| Column | Description |
|---|---|
| **RFI Number** | Sequential unique identifier (e.g., `RFI-001`) |
| **Subject** | Brief description of the information being requested |
| **Status** | Current status of the RFI |
| **Priority** | Urgency level |
| **Due Date** | Deadline by which a response is expected |
| **Assignee** | Person or team responsible for responding |
| **Date Submitted** | When the RFI was raised |
| **Date Responded** | When the response was provided (if applicable) |

## RFI Statuses

| Status | Meaning |
|---|---|
| **OPEN** | RFI has been submitted and awaits a response |
| **RESPONDED** | A response has been provided but the RFI is pending acknowledgement |
| **CLOSED** | The requester has accepted the response and the RFI is resolved |
| **OVERDUE** | The due date has passed without a response |

## Priority Levels

| Priority | Description |
|---|---|
| **LOW** | Information needed for future reference, no immediate impact |
| **MEDIUM** | Information needed within the normal workflow timeline |
| **HIGH** | Urgent — blocking or impacting current work |
| **CRITICAL** | Immediate response required — work is stopped pending this information |

## Creating an RFI

1. Click **Add RFI** (or **New RFI**)
2. Fill in:

| Field | Required | Description |
|---|---|---|
| **Subject** | Yes | Clear, specific description of the information needed |
| **Description** | Yes | Detailed explanation with context, references to drawings or specs |
| **Priority** | Yes | Select the urgency level |
| **Due Date** | Yes | Deadline for response |
| **Assignee** | Yes | Person or team expected to respond |
| **Attachments** | No | Supporting documents, photos, or drawing markups |

3. Click **Submit**

## RFI Best Practices

- **Be specific** — Reference drawing numbers, specification clauses, and locations
- **Set realistic due dates** — Account for review complexity
- **Attach context** — Include marked-up drawings or photos showing the issue
- **Follow up on overdue RFIs** — The system highlights overdue items automatically
