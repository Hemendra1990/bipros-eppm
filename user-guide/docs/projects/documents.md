# Documents

The **Documents** module provides a project-level document library for storing and organizing project files.

## Accessing Documents

Navigate to a project and click the **Documents** tab.

![Documents](/img/screenshots/25-project-documents.png)

## Features

### Folder Hierarchy

Documents are organized in a hierarchical folder structure. You can create nested folders to organize files by category:

- **Design Documents** — Architectural, structural, and MEP drawings
- **Contracts** — Contract agreements, amendments, and correspondence
- **Reports** — Monthly progress reports, site inspection reports
- **Approvals** — Regulatory approvals, environmental clearances
- **Correspondence** — Letters, emails, meeting minutes

### Document Metadata

Each uploaded document includes:

| Field | Description |
|---|---|
| **File Name** | The original name of the uploaded file |
| **Folder** | The folder location within the project's document tree |
| **Version** | Current version number |
| **Uploaded By** | The user who uploaded the document |
| **Upload Date** | Date and time of upload |
| **Description** | Optional description of the document's content |

### Version Control

When you upload a new version of an existing document, the system maintains version history. You can:

- View all previous versions
- Download any specific version
- Compare version dates and uploaders
